# 三阶段实施路线

不要一开始实现完整治理平台。每个阶段都必须先证明任务质量和维护成本的变化，再进入下一阶段。

## 第一阶段：低摩擦基础版

范围：Orcas Skill、`inbox`、`sources`、`knowledge`、`work`、`draft`、`trusted`、来源封存、带引用草稿、人工确认重要结论。

暂不做：复杂 Amend、并发锁、多层目录、自动发布、完整对话归档。

进入条件：有至少一个真实任务流和一组可以追溯的来源。退出条件：用户能只通过自然语言完成收集，且 trusted 内容没有被自动覆盖。

## 第二阶段：上下文增强版

范围：任务上下文包、事实与决策经验分离、相关性排序、过期提醒、冲突检测、任务结束后的增量沉淀。

进入条件：基础版已经持续使用，并能记录任务是否实际使用了注入上下文。退出条件：上下文命中率、错误引用和重复提问率有可比较的试点记录。

## 第三阶段：可选 Governed 模式

范围：Action Gateway、路径和状态强制校验、并发冲突检测、操作日志、健康报告、复用率和命中率统计。

进入条件：个人模式真实出现多 Agent 冲突、来源被覆盖风险或审计需求，而不是因为“系统看起来应该更完整”。

退出条件：治理带来的节省时间持续高于维护时间，并且所有受保护操作都有明确边界、验证方式和回滚路径。

## 统一评价

不要用文件数、规则数或目录完整度评价阶段成功。优先记录任务成功率、重复提问率、来源定位时间、上下文命中率、错误知识引用、人工确认比例、知识复用率、过期知识比例和系统维护耗时。
